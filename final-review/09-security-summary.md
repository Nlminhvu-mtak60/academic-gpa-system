# Security Summary — Protections, Authentication, & Access

Security reviews confirm that the Academic GPA Management System meets industrial safety standards.

---

## 1. Authentication & JWT Tokens

- **Access Tokens**: Short-lived (15 minutes) JWTs signed with `HMAC-SHA256` keys.
- **Refresh Tokens**: Saved in database with creation, expiration, and revocation logs. Refresh tokens are exchanged on secure HTTPS endpoints to fetch new short-lived access tokens.
- **Payload Safety**: Token claims only contain the unique user identifier and role assignments (`Student` or `Admin`). Sensitive details are never included.

---

## 2. Password Protection

- **Cryptographic Hasher**: Passwords are hashed using `PBKDF2` with `HMAC-SHA256` iterations and unique salts.
- **Salts**: Every password gets a unique random salt generated by cryptographically secure random number generators (`RandomNumberGenerator`).
- **Validation**: Rejects weak passwords that do not meet length limits.

---

## 3. Data Protection & Input Sanitization

- **SQL Injection Prevention**:
  - The application uses Entity Framework Core 10.0 for database operations.
  - EF Core utilizes parameterized queries, preventing SQL injection vulnerabilities.
- **XSS Protections**:
  - Inputs are validated against maximum character lengths (e.g., maximum 50-100 characters for names, 500 for comments).
  - React client automatically escapes variables rendered in the JSX DOM tree.
- **CORS Configuration**:
  - Production deployments use Nginx to proxy backend calls on the same origin domain.
  - The backend defines explicit origin-allow lists, preventing unauthorized external websites from calling the API.

---

## 4. Role-Based Access Controls (RBAC)

- **Endpoint Protection**: Controllers and MediatR handlers enforce access controls:
  - Users can only fetch and update their own academic profile data, semesters, course records, and AI advisor sessions.
  - Administrators have specific administrative access to lock/unlock users, reset passwords, and read server logs.
- **Database Filters**: EF Core filters enforce tenant separation at the query execution level, ensuring student actions never leak data to other accounts.
